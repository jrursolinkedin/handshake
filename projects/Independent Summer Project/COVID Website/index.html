How Naive Bayes Works

To understand the algorithm, it is best to go over an example of performing Naive Bayes on a dataset. By looking at an example, you will have an easier time implementing the algorithm. However, before going over an example, let's look at the Naive Bayes equation. Below is an the equation and the meaning of the terms.

P(A|C) = P(C|A)P(A) / P(C)

P(A|C): "Posterior Probability" or the probability of 'A' occuring given the evidence 'C' has alrady occured.
P(C|A): "Likelihood" or the probability of 'C' occuring given the evidence 'A' has already occured.
P(A): "Prior Probability" or the probabilty of 'A' occuring.
P(C): "Normalizing Constant" or the probability of 'C' occuring.

Example: From the goft dataset below, use Naive Bayes to determine the probabilites of playing and not playing golf given that the outlook is sunny, temperature is cool, humidity is high, and windy is strong.

Outlook Temperature Humidity Windy Play
sunny hot high false no
sunny hot high true no
overcast hot high false yes
rainy mild high false yes
rainy cool normal false yes
rainy cool normal true no
overcast cool normal true yes
sunny mild high false no
sunny cool normal false yes
rainy mild normal false yes
sunny mild normal true yes
overcast mild high true yes
overcast hot normal false yes
rainy mild high true no
STEP 1: First, let's find the prior probability, P(A),
P(Play = "yes") = 9/14
P(Play = "no") = 5/14
Just count the number of "yes" and "no" values within the class attribute, "Play", and divide those counts by the number of total values.
STEP 2: Next, find the likelihood of playing golf, P(C|A),
P(Outlook = "sunny" | Play = "yes") = 2/9
P(Temperature = "cool" | Play = "yes") = 3/9
P(Humidity = "high" | Play = "yes") = 3/9
P(Windy = "strong" | Play = "yes") = 3/9
For each attribute, count the number of times "yes" shows in the class attribute.
STEP 3: Similarly, find the likelihood of not playing golf, P(C|A),
P(Outlook = "sunny" | Play = "no") = 3/5
P(Temperature = "cool" | Play = "no") = 1/5
P(Humidity = "high" | Play = "no") = 4/5
P(Windy = "strong" | Play = "no") = 3/5
For each attribute, count the number of times "no" shows in the class attribute.
STEP 4: Now, we combine the three previous steps, P(X|C)*P(C),
P(attributes | Play = "yes") = (2/9)*(3/9)*(3/9)*(3/9)*(9/14) = 0.0053
P(attributes | Play = "no") = (3/5)*(1/5)*(4/5)*(3/5)*(5/14) = 0.0206
We are just multiplying all the values we got from the "yes"'s and "No"'s in the previous steps.
STEP 5: Lastly, we need to find the normalizing constant, P(C),
P(C) = P(OutLook = "sunny")*P(Temperature = "cool")*P(Humidity = "high")*P(Wind = "strong")
P(C) = (5/14)*(4/14)*(7/14)*(6/14)
P(C) = 0.02186
Simply count the number of times each attribute shows up, and divide those counts by the number of total values. Then multiply all the probabilities together to get P(C).
STEP 6: Finally, let's calculate the posterior probability, P(A|C),
P(Play = "yes" | C) = 0.0053/0.02186 = 0.2424
P(Play = "no" | C) = 0.0206/0.02186 = 0.9421
We just divide P(X|C)*P(C) by the normalizing constant.

Answer: From performing Naive Bayes on the given attribute values, the probability of playing golf is 24.24% and the probability of not playing golf is 94.21%. Thus, if this was a real life scenario, we would likely conclude that golf would not be played.
